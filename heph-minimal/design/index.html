Tutorial: DESTROY ALL POCKET MONSTERS!

Hephaestus uses my scripting language Minimal and a library of scripting functions to create adventures.

The entire adventure is specified in a Minimal script, which should have the ".min" extension. When you're ready to try it out, run the compiler on your script to produce a ".adv" file.

When Hephaestus loads an adventure, it executes the 'newgame()' function, which you must define. The rest is up to you; you can do anything you like from that function. The only thing that exists before the script starts is the player, with id "player". When the 'newgame()' function returns, the game starts.

Note that you cannot have any immediate commands in your script that call Hephaestus, or the compiler will reject your script - you should only have 'def()' statements.

When the game is saved, all of the global variables are also saved, so you can use them to maintain game state, particularly quest flags (but see also 'st_prop()', which is usually a better solution).

Example

  1. Create a new text file 'example.min', and type in the following: 
    def(newgame())
    output("The smallest Hephaestus adventure ever!")
    map_new("test_M", "Test Map", 11, 11)
    th_moveto("player", "test_M", 5, 5)
    ;
  2. Now compile it by running 'compile.sh' (Unix) or 'compile.bat' (Windows) and picking 'example.min' in the file selector. This will create 'example.adv'.
  3. Start Hephaestus by running 'heph.sh' or 'heph.bat', choose "New Adventure" from the "Game" menu, and pick 'example.adv'.
  4. The output window will show: 
    The smallest Hephaestus adventure ever!

    In the view, you will see the player standing in the middle of a black void (because you haven't set any tiles). It works!

Read the tutorial or sample.min to see a more complete and commented example, and then read the other design topics.
